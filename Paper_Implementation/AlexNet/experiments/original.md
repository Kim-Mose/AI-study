# 논문 결과 재현 실험

## 목표
AlexNet의 핵심 구조를 PyTorch로 구현하여 분류 성능을 확인한다.<br>
원본은 ImageNet (1000 클래스, 120만 장)이지만 자원 한계로 **CIFAR-10 일부**로 진행.

## 실험 설정

### 데이터셋
- **CIFAR-10** (32×32 → 224×224 리사이즈)
- 학습: 5,000장 (서브셋, 자원 제약)
- 테스트: 1,000장
- ⚠️ 데모 실행이며 전체 데이터 + 더 긴 epoch이 권장됨

### 모델
원본 AlexNet 구조 (출력층만 1000 → 10 변경)

### 하이퍼파라미터
| 항목 | 값 |
| --- | --- |
| Optimizer | Adam |
| Learning Rate | 0.001 |
| Batch Size | 64 |
| Epochs | 3 |

### 환경
- Mac mini (MPS)

## 실험 결과

### 학습 결과
| Epoch | Train Loss | Train Acc | Test Loss | Test Acc |
| --- | --- | --- | --- | --- |
| 1 | 2.1681 | 0.1792 | 2.1536 | 0.2360 |
| 2 | 1.9777 | 0.2568 | 1.9031 | 0.2940 |
| 3 | 1.8319 | 0.3078 | 1.8762 | 0.3300 |

### 최종 성능
- **테스트 정확도: 33.00%** (3 epoch, 5000장 subset)
- 참고 - ImageNet 원본: Top-5 에러 15.3%

## 분석

### 데모 환경의 한계
- **전체 데이터(50,000장)와 충분한 epoch(20+)이 필요**
- 5000장 + 3 epoch은 모델이 학습을 시작하는 단계
- 학습 곡선이 우상향 → 더 학습하면 성능 향상 가능
- ImageNet 같은 대규모 데이터셋에 맞춘 모델이라 작은 데이터에선 비효율적

### 관찰된 패턴
- Loss는 매 epoch마다 감소 → 모델이 학습 중
- Test Acc가 Train Acc보다 높음 → 아직 과적합 단계 도달 안 함
- 첫 epoch부터 17.92% (랜덤 10%보다 높음) → 학습 진행 중

## 결론
AlexNet 구조가 정상 작동함을 확인.<br>
CIFAR-10 같은 작은 이미지(32×32)에는 11×11 필터가 비효율적임을 시사.<br>
완전한 평가를 위해서는 ImageNet 또는 전체 CIFAR-10 + 충분한 epoch 필요.
