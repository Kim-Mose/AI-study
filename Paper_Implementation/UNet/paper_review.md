# U-Net 논문 리뷰

## 기본 정보
- **제목**: U-Net: Convolutional Networks for Biomedical Image Segmentation
- **저자**: Olaf Ronneberger, Philipp Fischer, Thomas Brox
- **발표**: MICCAI 2015
- **링크**: https://arxiv.org/abs/1505.04597

## 한 줄 요약
의료 영상 세그멘테이션을 위해 인코더-디코더 + 스킵 연결 구조를 제안한 모델.

## 배경
의료 영상은:
- 데이터가 적음 (수백~수천 장)
- 픽셀 단위로 정확한 분류 필요 (분류가 아니라 세그멘테이션)
- 기존 CNN은 이미지 전체를 분류만 가능, 픽셀 단위 예측 어려움

기존 방식인 sliding window 방식은:
- 느림
- 컨텍스트와 정확도 사이의 트레이드오프

## 핵심 아이디어

### 1. 인코더-디코더 구조
```
인코더 (수축 경로): 점점 작아지면서 추상적 특성 추출
디코더 (확장 경로): 점점 커지면서 위치 정보 복원
```

### 2. 스킵 연결
인코더의 특성을 디코더에 직접 전달<br>
→ 다운샘플링으로 잃은 정보를 복원

### 3. U자 모양
구조가 U자 모양이라 U-Net이라는 이름.<br>
좌측: 다운샘플링, 우측: 업샘플링

## 모델 구조

### 인코더 (수축 경로)
- DoubleConv (3×3 Conv + ReLU 두 번)
- MaxPool 2×2
- 채널 수: 64 → 128 → 256 → 512

### Bottleneck
- 가장 깊은 부분: DoubleConv (1024 채널)

### 디코더 (확장 경로)
- ConvTranspose 2×2 (업샘플링)
- 인코더의 같은 레벨과 concat (스킵 연결)
- DoubleConv

### 출력
- 1×1 Conv로 클래스 수만큼 채널 만들기

## 학습 방법
- **손실 함수**: Cross-Entropy + Pixel-wise softmax
- **데이터 증강**: 회전, 변형, 탄성 변형 (의료 영상 특성)
- **가중치 부여**: 경계 부분에 더 큰 가중치 (분리된 객체 구분)

## 실험 결과
의료 영상 세그멘테이션 챌린지에서 SOTA 달성.

## 한계점
1. **메모리 사용량 큼** - 인코더 특성 저장 필요
2. **이미지 크기에 따라 출력 크기 다름** - 최근에는 padding으로 해결
3. **3D 데이터에는 별도의 3D U-Net 필요**

## 느낀 점

### 1. 단순하지만 강력한 구조
인코더-디코더 + 스킵 연결이라는 구조가 매우 효과적임을 입증.<br>
이 구조는 이후 거의 모든 세그멘테이션 모델의 기본이 됨.

### 2. 도메인 특화의 중요성
의료 영상이라는 특수한 도메인에 맞춰 설계된 모델.<br>
"범용"이 아닌 "특화"의 가치를 보여줌.

### 3. 적은 데이터에서의 강점
ImageNet급 데이터가 없어도 잘 동작.<br>
이는 강력한 데이터 증강과 구조의 효율성 덕분.

## 응용 분야
- 의료 영상 (CT, MRI 분할)
- 위성 이미지 분석
- 자율 주행 (도로/차선 분할)
- 일반 객체 세그멘테이션

## 면접 질문

**Q1: U-Net의 핵심 아이디어는?**<br>
A: 인코더-디코더 구조 + 스킵 연결. 인코더로 추상적 특성을 추출하고,<br>
디코더로 위치 정보를 복원하며, 스킵 연결로 정보 손실을 방지한다.

**Q2: 왜 의료 영상에서 잘 동작하는가?**<br>
A: 1) 적은 데이터로 학습 가능. 2) 픽셀 단위 정확도. 3) 강력한 데이터 증강 적용.

**Q3: 분류 모델과의 차이는?**<br>
A: 분류는 이미지 → 단일 라벨. 세그멘테이션은 이미지 → 픽셀별 라벨.<br>
U-Net은 모든 픽셀에 대해 클래스를 예측한다.
